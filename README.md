任务数据面板

项目简介
这是一个基于 JavaScript 实现 API 交互与数据动态渲染的任务管理面板。页面在静态 HTML/CSS 的基础上，通过调用后端 RESTful API，完整实现了任务的获取、创建、筛选与状态管理功能。

我完成的功能
1.  获取任务列表：页面加载后自动从 API 获取所有任务并渲染。
2.  刷新任务列表：点击“刷新任务”按钮，重新请求并更新列表。
3.  提交新任务：通过表单填写任务信息，提交后可创建新任务并实时更新列表。
4.  按状态筛选任务：通过下拉框可选择“全部”、“待开始”、“进行中”或“已完成”状态，页面列表会动态过滤显示。
5.  错误提示：在网络请求失败或表单提交出错时，页面会给出明确的中文错误提示。

我的实现思路
1. 页面加载与初始化
在 `DOMContentLoaded` 事件中，调用 `fetchTasks()` 函数。该函数使用 `fetch()` 方法向 `GET /api/tasks` 接口发起请求，获取初始任务数据，并将返回的 JSON 数据保存在全局变量 `allTasks` 中，供后续筛选使用。

2. 获取和渲染数据
- 获取：`fetchTasks()` 函数处理了网络请求和响应解析。使用 `async/await` 语法让代码更清晰。
- 渲染：定义 `renderTasks(tasks)` 函数。它首先清空 `<ul id="taskList">` 容器，然后遍历传入的任务数组，为每个任务动态创建 `<li class="task-item">` DOM 元素，并将任务标题、负责人、状态（转换为中文并添加对应 CSS 类）和创建时间格式化后插入其中，最后将任务项添加到列表容器。

3. 提交新任务
- 为表单 `<form id="taskForm">` 绑定 `submit` 事件监听。
- 在事件处理函数 `handleFormSubmit(event)` 中，阻止表单默认提交行为，使用 `FormData` 对象收集输入框和下拉框的数据。
- 对数据进行简单的前端验证后，使用 `fetch()` 方法向 `POST /api/tasks` 接口发送请求，请求体为 JSON 格式的任务数据。
- 收到成功响应后，在页面上显示成功提示，清空表单输入框，并自动调用 `fetchTasks()` 刷新任务列表。

4. 状态筛选
- 为下拉框 `<select id="statusFilter">` 绑定 `change` 事件监听。
- 在事件处理函数中，读取当前选中的值，与全局变量 `allTasks` 进行比对。利用数组的 `filter()` 方法，筛选出状态匹配的任务（或选择“全部”时返回所有任务），然后调用 `renderTasks()` 函数重新渲染筛选后的结果。

5. 错误处理与用户体验
- 在所有 `fetch()` 请求外围使用 `try...catch` 进行包裹，捕获网络错误。
- 检查响应的 `ok` 状态及接口返回的 `success` 字段，在失败时抛出错误。
- 在 `catch` 块中，将错误信息显示在页面预留的 `<p class=“message”>` 区域，并为成功、失败、加载中等不同状态设置了不同的文字颜色（如红色表错误，绿色表成功）。
- 在提交和加载数据时，在消息区域显示“提交中…”、“加载任务列表中…”等加载状态提示。
